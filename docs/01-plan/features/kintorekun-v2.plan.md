## 1. 배경 및 목적

### 1.1 현재 상황
킨토레쿤은 개인 운동 기록/루틴 관리 앱으로, 현재:
- 운동 기록 저장 및 히스토리 조회
- 일일 루틴 체크리스트
- 월간 달력 플랜
- 기본 권장 무게 계산 (체중 × 계수)

### 1.2 문제점
| 문제 | 영향 |
|------|------|
| 운동 플랜을 매번 사용자가 직접 설계해야 함 | 귀찮아서 포기하거나 편향된 운동(예: 가슴만 계속) |
| 어느 근육군이 부족한지 직관적으로 모름 | Push/Pull/Legs 불균형 → 부상 위험 |
| 혼자 하는 운동의 동기 부여 부족 | 꾸준함 유지 어려움 |
| 근처 헬스장 찾을 때 앱을 벗어나야 함 | 앱 이탈, 사용성 저하 |

### 1.3 목표
- 기록 데이터를 활용한 **지능형 주간 플랜 자동 생성**
- 위치 기반 **헬스장 탐색 및 체크인** 기능으로 앱 내 완결
- 커뮤니티 랭킹으로 **사회적 동기 부여** 시스템 구축

## 2. 기능 요구사항

### Feature 1: 알고리즘 기반 주간 운동 플랜 추천

#### 핵심 로직
```
최근 28일 기록 분석
    → Push / Pull / Legs 빈도 집계
    → 가장 부족한 근육군 파악
    → PPL 스플릿 템플릿 선택 (3종)
    → 각 운동별 점진적 과부하 계산
    → 2.5kg 단위 반올림 + 안전 상한선 적용
    → 7일 플랜 카드 렌더링
```

#### 사용자 스토리
- 신규 사용자: 체중 기반 기본 무게 추천 (기록 없어도 동작)
- 기존 사용자: 최근 기록 분석 → 무게 자동 증량 추천
- "새로고침" 버튼으로 언제든 재계산 가능

#### 점진적 과부하 규칙
| 조건 | 무게 변화 |
|------|-----------|
| 최근 7일 동일 운동 3회 이상 | +2.5% |
| 최근 28일 동일 운동 9회 이상 | +5% |
| 기록 없음 | 체중 × 기본 계수 |

---

### Feature 2: 위치 기반 근처 헬스장 목록

#### 사용자 플로우
```
"주변 헬스장 검색" 버튼 클릭
    → 브라우저 위치 권한 요청
    → 10분 캐시 확인 (있으면 즉시 표시)
    → Overpass API 호출 (2km 반경)
    → 거리순 정렬 → 텍스트 목록 표시
    → "체크인" 클릭 → 체크인 상태 저장
    → 커뮤니티 랭킹 섹션 표시
```

#### 데이터 구조 (localStorage)
```js
'kintore-checkin-v5'   → { gymName, gymId, checkinTime, lat, lon }
'kintore-gym-cache-v5' → { lat, lon, timestamp, gyms[] }
```

#### 제약 사항
- 지도 표시 없음 (텍스트 목록만)
- Overpass API 무료 사용 (rate limit 고려 → 10분 캐시)
- 이름 없는 장소는 목록에서 제외

---

### Feature 3: 커뮤니티 헬스장 랭킹 (Firebase)

#### Firestore 데이터 구조
```
gyms/{gymId}/
  name, lat, lon, createdAt
  leaderboard/
    squat    → { entries: [{nickname, weight, submittedAt, badge}] }
    deadlift → { entries: [...] }
    bench    → { entries: [...] }
```

#### 뱃지 시스템
| 1위 종목 | 뱃지 |
|----------|------|
| 스쿼트 | 스쿼트 짱 |
| 데드리프트 | 데드리프트 짱 |
| 벤치프레스 | 벤치프레스 짱 |

#### 규칙
- 닉네임당 1개 기록만 유지 (갱신 시 덮어쓰기)
- 랭킹 최대 20개 보존 (초과 시 하위 기록 삭제)
- Firestore Transaction으로 동시 제출 정합성 보장

#### 선행 조건
> 사용자가 Firebase 콘솔에서 프로젝트 생성 후 `firebase-config.js` 직접 작성 필요
> Firebase 없을 경우 안내 메시지 표시 (앱 오류 없음)

## 5. 검증 방법

### Feature 1 검증
- [ ] 운동 기록 없는 신규 상태 → 기본 추천 플랜 표시
- [ ] 운동 기록 추가 후 → 점진적 과부하 무게 반영 확인
- [ ] 특정 근육군 집중 기록 후 → 반대 근육군 더 많이 배치 확인

### Feature 2 검증
- [ ] 브라우저에서 위치 허용 → 목록 표시
- [ ] 체크인 → 새로고침 후 체크인 상태 유지
- [ ] 10분 내 재검색 → 캐시 사용 (API 미호출)

### Feature 3 검증
- [ ] Firebase 없이 접속 → 안내 메시지 표시 (오류 없음)
- [ ] Firebase 설정 후 기록 등록 → Firebase 콘솔에서 데이터 확인
- [ ] 2개 브라우저 동시 제출 → 트랜잭션 정합성 확인
